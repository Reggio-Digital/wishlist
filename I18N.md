# Multi-Language Support (i18n)

The Wishlist App supports multiple languages through internationalization (i18n) powered by `next-intl`.

## Supported Languages

The following languages are currently supported:

- **English** (`en`) - Default
- **Spanish** (`es`)
- **French** (`fr`)
- **German** (`de`)

## Configuration

### Setting the Default Language

You can configure the default language for your application using the `DEFAULT_LANGUAGE` environment variable in Docker.

#### Docker Compose

Add or modify the `DEFAULT_LANGUAGE` variable in your `.env` file or `docker-compose.yml`:

```yaml
environment:
  - DEFAULT_LANGUAGE=es  # Options: en, es, fr, de
```

Or in your `.env` file:

```
DEFAULT_LANGUAGE=es
```

If not set, the application defaults to English (`en`).

### Accessing Different Languages

Users can access the application in different languages by adding a locale prefix to the URL:

- English: `https://your-domain.com/` or `https://your-domain.com/en`
- Spanish: `https://your-domain.com/es`
- French: `https://your-domain.com/fr`
- German: `https://your-domain.com/de`

The default language is accessible both with and without the locale prefix.

## Adding New Languages

To add support for a new language:

1. **Create a translation file**: Add a new JSON file in `frontend/messages/` named with the language code (e.g., `it.json` for Italian).

   ```json
   {
     "common": {
       "appName": "App Lista dei Desideri",
       ...
     },
     ...
   }
   ```

2. **Update the i18n configuration**: Add the new locale to `frontend/i18n.ts`:

   ```typescript
   export const locales = ['en', 'es', 'fr', 'de', 'it'] as const;
   ```

3. **Rebuild the Docker container**:

   ```bash
   docker-compose down
   docker-compose build frontend
   docker-compose up -d
   ```

## Translation Structure

Translation files are organized into namespaces for better organization:

- `common`: Shared text across the application
- `home`: Homepage-specific text
- `admin`: Admin panel text
- `wishlist`: Wishlist management
- `item`: Item management
- `claims`: Claims functionality
- `settings`: Settings page
- `validation`: Form validation messages
- `errors`: Error messages

## Development

### Using Translations in Components

To use translations in your components:

```tsx
import { useTranslations } from 'next-intl';

export default function MyComponent() {
  const t = useTranslations('namespace');

  return <h1>{t('key')}</h1>;
}
```

### Testing Different Languages

During development, you can test different languages by navigating to:

- `http://localhost:3001/en`
- `http://localhost:3001/es`
- `http://localhost:3001/fr`
- `http://localhost:3001/de`

## Notes

- The language setting affects the entire frontend application
- Users can manually switch languages by changing the URL locale prefix
- The default language is used for users who don't specify a locale in the URL
- All translations are loaded at build time for optimal performance
